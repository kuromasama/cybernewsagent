# 🚨 解析 TeamPCP 供應鏈攻擊：LiteLLM 後門分析與防禦策略

> **⚡ 戰情快篩 (TL;DR)**
> * **嚴重等級**: Critical (CVSS: 9.8)
> * **受駭指標**: RCE (Remote Code Execution) 和 Info Leak
> * **關鍵技術**: `Supply Chain Attack`, `Package Hijacking`, `PythonPTH`

## 1. 🔬 漏洞原理與技術細節 (Deep Dive)

* **Root Cause**: LiteLLM 套件在 PyPI 上的版本 1.82.7 和 1.82.8 被竄改，包含了一個名為 `proxy_server.py` 的後門檔案。這個檔案在載入時會執行隱藏的酬載，進行三階段攻擊：搜刮憑證、橫向移動和安裝後門程式。
* **攻擊流程圖解**:
  1. 使用者安裝受駭的 LiteLLM 版本。
  2. `proxy_server.py` 後門檔案被載入並執行。
  3. 攻擊者搜刮憑證，包括 SSH 金鑰、雲端權杖和 Kubernetes 密鑰等。
  4. 攻擊者在 Kubernetes 叢集中橫向移動和安裝後門程式。
  5.竊取的資料被加密處理並回傳至攻擊者控制的伺服器。
* **受影響元件**: LiteLLM 版本 1.82.7 和 1.82.8，Python 3.x。

## 2. ⚔️ 紅隊實戰：攻擊向量與 Payload (Red Team Operations)

* **攻擊前置需求**: 需要有權限安裝 Python 套件，且目標環境需要使用 LiteLLM。
* **Payload 建構邏輯**:

    ```

    import requests

        # ...
    
    #加密處理和回傳竊取的資料
    def exfiltrate_data():
        # ...
    
    #載入並執行隱藏的酬載
    def load_payload():
        # ...
    
    load_payload()
    
    ```
* **範例指令**: 使用 `curl` 下載和安裝受駭的 LiteLLM 版本。

```

bash
curl -s https://pypi.org/packages/lite-llm-1.82.7.tar.gz | tar -xvf -
python setup.py install

```
* **繞過技術**: 可以使用 WAF 繞過技巧，例如使用 Base64 編碼的 Payload。

## 4. 📚 專有名詞與技術概念解析 (Technical Glossary)

* **Supply Chain Attack (供應鏈攻擊)**: 想像一個公司的供應鏈，如同一條長長的鏈子，攻擊者可以在鏈子的任何一環上發動攻擊。技術上是指攻擊者竄改或操控軟件供應鏈中的某個環節，例如開源庫或第三方套件，以達到惡意目的。
* **Package Hijacking (套件劫持)**: 想像一個套件被劫持，攻擊者可以在其中添加惡意代碼。技術上是指攻擊者竄改或操控軟件套件，以達到惡意目的。
* **PythonPTH (Python 路徑)**: 想像 Python 的路徑，如同一張地圖，攻擊者可以在其中添加惡意路徑。技術上是指 Python 的路徑機制，可以用來載入和執行 Python 代碼。
