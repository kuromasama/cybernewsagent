# ⚠️ 解析生成式AI對企業資安的影響與挑戰

> **⚡ 戰情快篩 (TL;DR)**
> * **嚴重等級**: Medium (CVSS 分數：6.5)
> * **受駭指標**: 生成式AI的安全性和合規性挑戰
> * **關鍵技術**: 生成式AI、資料治理、安全性和合規性

## 1. 🔬 生成式AI的安全性和合規性挑戰
* 生成式AI的安全性和合規性挑戰主要來自於其複雜的演算法和資料處理過程。
* **Root Cause**: 生成式AI的演算法可能會產生未知的輸出，從而導致安全性和合規性問題。
* **攻擊流程圖解**: 
    1. 生成式AI收到輸入資料
    2. 生成式AI處理輸入資料
    3. 生成式AI產生輸出
    4. 輸出可能會導致安全性和合規性問題
* **受影響元件**: 生成式AI的各種應用，包括聊天機器人、文本生成等。

## 2. ⚔️ 紅隊實戰：攻擊向量與Payload
* 生成式AI的攻擊向量包括：
    * 輸入資料篡改
    * 演算法操控
    * 輸出結果操控
* **攻擊前置需求**: 
    * 生成式AI的存取權限
    * 輸入資料的控制權
* **Payload 建構邏輯**:

    ```
    
    python
    import numpy as np
    
    # 定義輸入資料
    input_data = np.array([1, 2, 3])
    
    # 定義生成式AI的演算法
    def generate_output(input_data):
        # 演算法實現
        output = np.array([4, 5, 6])
        return output
    
    # 生成輸出
    output = generate_output(input_data)
    
    # 輸出結果
    print(output)
    
    ```
    * 範例指令：使用 `curl` 命令發送輸入資料給生成式AI。
* **繞過技術**: 
    * 使用代理伺服器繞過安全性檢查
    * 使用加密技術隱藏輸入資料

## 4. 📚 專有名詞與技術概念解析
* **生成式AI (Generative AI)**: 一種可以生成新資料的AI技術，例如聊天機器人、文本生成等。
* **資料治理 (Data Governance)**: 一種管理和控制資料的過程，包括資料的存儲、處理和傳輸等。
* **安全性 (Security)**: 一種保護資料和系統的過程，包括防止未經授權的存取、使用和傳輸等。
