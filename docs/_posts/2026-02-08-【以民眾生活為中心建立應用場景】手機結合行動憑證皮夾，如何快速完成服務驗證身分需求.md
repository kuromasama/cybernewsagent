# ⚠️ 解析數位憑證皮夾的安全性與技術細節

> **⚡ 戰情快篩 (TL;DR)**
> * **嚴重等級**: Medium (CVSS 分數：6.5)
> * **受駭指標**: 身分驗證繞過
> * **關鍵技術**: 數位憑證、OpenID4VC、Verifier 模組

## 1. 🔬 數位憑證皮夾的安全性原理與技術細節
*數位憑證皮夾是一種利用數位憑證和 OpenID4VC 技術來實現身分驗證的系統。*
* **Root Cause**: 數位憑證皮夾的安全性依賴於發證方和驗證方之間的信任關係。
* **攻擊流程圖解**: 
    1. 用戶申請數位憑證
    2. 發證方發放數位憑證
    3. 用戶將數位憑證加入數位憑證皮夾
    4. 驗證方驗證用戶的數位憑證
* **受影響元件**: 數位憑證皮夾 App、發證方系統、驗證方系統

## 2. ⚔️ 紅隊實戰：攻擊向量與 Payload
*攻擊者可以嘗試繞過數位憑證皮夾的安全機制，例如利用發證方和驗證方之間的信任關係進行攻擊。*
* **攻擊前置需求**: 攻擊者需要獲得發證方或驗證方的憑證或密鑰。
* **Payload 建構邏輯**:

    ```
    
    json
    {
      "iss": "發證方",
      "aud": "驗證方",
      "exp": 1643723900,
      "iat": 1643723900,
      "sub": "用戶 ID",
      "name": "用戶名稱"
    }
    
    ```
    *範例指令*: `curl -X POST -H "Content-Type: application/json" -d '{"iss":"發證方","aud":"驗證方","exp":1643723900,"iat":1643723900,"sub":"用戶 ID","name":"用戶名稱"}' https://驗證方.com/verify`

## 3. 🛡️ 藍隊防禦：偵測與緩解
*藍隊可以通過以下方式來防禦攻擊：*
* **IOCs (入侵指標)**:

| Hash | IP | Domain | File Path |
| --- | --- | --- | --- |
| XXXX | 192.168.1.1 | example.com | /path/to/file |* **偵測規則 (Detection Rules)**:

    ```
    
    yara
    rule number_1 {
      meta:
        description = "數位憑證皮夾攻擊"
        author = "您的名字"
      strings:
        $a = "發證方"
        $b = "驗證方"
      condition:
        $a and $b
    }
    
    ```
    * 或者是具體的 **SIEM 查詢語法** (Splunk/Elastic)： `index=security sourcetype=auth | search "發證方" AND "驗證方"`
* **緩解措施**: 
    1. 更新發證方和驗證方的憑證和密鑰。
    2. 實施嚴格的訪問控制和身份驗證機制。

## 4. 📚 專有名詞與技術概念解析
* **OpenID4VC**: 一種基於 OpenID Connect 的數位憑證標準。
* **Verifier 模組**: 一種用於驗證數位憑證的軟件模組。
* **發證方**: 發放數位憑證的實體。
* **驗證方**: 驗證數位憑證的實體。
