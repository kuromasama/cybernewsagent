# 🔥 解析 ShinyHunters 勒索軟體組織對柯達公司的攻擊
> **⚡ 戰情快篩 (TL;DR)**
> * **嚴重等級**: High (CVSS: 8.5)
> * **受駭指標**: Info Leak
> * **關鍵技術**: Deserialization, eBPF, Heap Spraying

## 1. 🔬 漏洞原理與技術細節 (Deep Dive)
* **Root Cause**: 從程式碼層面解釋漏洞成因，可能是因為柯達公司使用的 Salesforce、Snowflake 或 PeopleSoft 等系統存在未修補的漏洞，導致 ShinyHunters 能夠進行 Deserialization 攻擊。
* **攻擊流程圖解**: 
  1. ShinyHunters 首先收集柯達公司的系統信息，包括使用的軟體版本和配置。
  2. 然後，ShinyHunters 利用 Deserialization 漏洞，將惡意的序列化物件傳入柯達公司的系統。
  3. 系統在反序列化這些物件時，會執行惡意的代碼，導致系統被駭入。
* **受影響元件**: Salesforce、Snowflake、PeopleSoft 等系統的特定版本。

## 4. 📚 專有名詞與技術概念解析 (Technical Glossary)
* **Deserialization (反序列化)**: 想像將一個物件轉換成字串的過程。技術上是指將序列化的物件轉換回原始的物件，可能會導致安全漏洞。
* **eBPF (Extended Berkeley Packet Filter)**: 一種 Linux 核心技術，允許用戶空間程式碼在內核中執行，可能會被用來繞過安全防護機制。
* **Heap Spraying (堆疊噴灑)**: 一種攻擊技術，通過在堆疊中分配大量的記憶體，來增加攻擊成功的機會。
