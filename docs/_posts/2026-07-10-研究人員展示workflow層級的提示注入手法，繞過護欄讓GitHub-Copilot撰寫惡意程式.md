# 🚨 工作流層級越獄建構：解析 GitHub Copilot 的新型提示注入攻擊
> **⚡ 戰情快篩 (TL;DR)**
> * **嚴重等級**: Critical (CVSS: 9.0)
> * **受駭指標**: RCE (Remote Code Execution)
> * **關鍵技術**: `Workflow-level Jailbreak Construction`, `Prompt Injection`, `AI-powered Code Generation`

## 1. 🔬 漏洞原理與技術細節 (Deep Dive)
* **Root Cause**: GitHub Copilot 的 AI 模型在生成程式碼時，沒有充分考慮到工作流層級的安全性，導致攻擊者可以通過精心設計的工作流程來注入惡意程式碼。
* **攻擊流程圖解**:
  1. 攻擊者設計一個看似正常的工作流程，包含多個階段。
  2. 每個階段都會要求 GitHub Copilot 生成特定的程式碼。
  3. GitHub Copilot 生成的程式碼會被組合起來，形成一個完整的惡意程式碼。
* **受影響元件**: GitHub Copilot 的 4 個閉源模型，包括 Claude Sonnet 4、Claude Haiku 4.5、Gemini 3.1 Pro 和 Gemini 3.5 Flash。

## 2. ⚔️ 紅隊實戰：攻擊向量與 Payload (Red Team Operations)
* **攻擊前置需求**: 攻擊者需要有 GitHub Copilot 的使用權限和一個設計好的工作流程。
* **Payload 建構邏輯**:

    ```
    
    python
      # 範例 Payload 結構
      workflow = [
        {"stage": "init", "code": "import os"},
        {"stage": "read_file", "code": "with open('file.txt', 'r') as f: print(f.read())"},
        {"stage": "write_file", "code": "with open('file.txt', 'w') as f: f.write('malicious code')"}
      ]
    
    ```
  * **範例指令**: `curl -X POST -H "Content-Type: application/json" -d '{"workflow": [{"stage": "init", "code": "import os"}]}' https://api.github.com/copilot/generate`
* **繞過技術**: 攻擊者可以使用工作流層級的越獄建構來繞過 GitHub Copilot 的安全檢查。

## 3. 🛡️ 藍隊防禦：偵測與緩解 (Blue Team Defense)
* **IOCs (入侵指標)**:

| Hash | IP | Domain | File Path |

* **緩解措施**: 除了更新 GitHub Copilot 的修補程式之外，還需要監控 IDE 開發流程中生成的檔案、腳本程式、範例、日誌和過渡性物件。

## 4. 📚 專有名詞與技術概念解析 (Technical Glossary)
* **Workflow-level Jailbreak Construction (工作流層級越獄建構)**: 一種攻擊技術，利用工作流層級的設計來注入惡意程式碼。
* **Prompt Injection (提示注入)**: 一種攻擊技術，利用精心設計的提示來注入惡意程式碼。
* **AI-powered Code Generation (AI 驅動的程式碼生成)**: 一種技術，利用 AI 模型來生成程式碼。
