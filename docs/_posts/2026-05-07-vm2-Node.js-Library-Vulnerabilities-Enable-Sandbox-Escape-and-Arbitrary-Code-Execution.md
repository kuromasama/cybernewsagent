# 🚨 解析 Node.js vm2 庫的多個高風險漏洞：沙盒逃逸與任意代碼執行

> **⚡ 戰情快篩 (TL;DR)**
> * **嚴重等級**: Critical (CVSS 分數最高達 10.0)
> * **受駭指標**: RCE (Remote Code Execution)
> * **關鍵技術**: `Sandbox Escape`, `JavaScript Injection`, `Prototype Pollution`

## 1. 🔬 漏洞原理與技術細節 (Deep Dive)

* **Root Cause**: vm2 庫的沙盒機制存在多個漏洞，允許攻擊者通過特定的 JavaScript 代碼逃逸沙盒，執行任意代碼。這些漏洞主要是由於對 JavaScript 物件的代理和攔截機制存在缺陷，導致攻擊者可以操控沙盒環境中的物件，進而突破沙盒限制。
* **攻擊流程圖解**:
  1. 攻擊者構造特殊的 JavaScript 代碼，利用漏洞逃逸沙盒。
  2. 代碼被執行於沙盒環境中，利用漏洞操控沙盒中的物件。
  3. 攻擊者通過操控物件，突破沙盒限制，執行任意代碼。
* **受影響元件**: vm2 庫版本 <= 3.10.5，Node.js 版本 25.6.1。

## 2. ⚔️ 紅隊實戰：攻擊向量與 Payload (Red Team Operations)

* **攻擊前置需求**: 攻擊者需要能夠執行任意 JavaScript 代碼於沙盒環境中。
* **Payload 建構邏輯**:

    ```
    
    javascript
      // 範例 Payload
      const payload = {
        "__proto__": {
          "toString": function() {
            // 任意代碼執行
            require("child_process").exec("ls -l");
          }
        }
      };
    
    ```
 

```

bash
  # 範例指令
  curl -X POST \
    http://example.com/vm2 \
    -H 'Content-Type: application/json' \
    -d '{"__proto__": {"toString": "require(\"child_process\").exec(\"ls -l\")"}}'

```
* **繞過技術**: 攻擊者可以使用各種繞過技術，例如使用 `eval()` 函數或 `Function` 建構函數來執行任意代碼。

## 4. 📚 專有名詞與技術概念解析 (Technical Glossary)

* **Sandbox Escape**: 想像一個應用程序在一個安全的環境中運行，攻擊者試圖突破這個環境的限制，執行任意代碼。技術上是指攻擊者利用漏洞或其他手段，逃逸沙盒環境，執行任意代碼。
* **JavaScript Injection**: 想像一個攻擊者將惡意的 JavaScript 代碼注入到一個應用程序中，技術上是指攻擊者利用漏洞或其他手段，將惡意的 JavaScript 代碼注入到一個應用程序中，執行任意代碼。
* **Prototype Pollution**: 想像一個攻擊者操控一個物件的原型，技術上是指攻擊者利用漏洞或其他手段，操控一個物件的原型，執行任意代碼。
