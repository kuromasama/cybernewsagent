# ⚠️ 解析 Apple 帳號變更通知釣魚攻擊
> **⚡ 戰情快篩 (TL;DR)**
> * **嚴重等級**: Medium (CVSS: 6.1)
> * **受駭指標**: Info Leak
> * **關鍵技術**: `SPF`, `DKIM`, `DMARC`

## 1. 🔬 漏洞原理與技術細節 (Deep Dive)
* **Root Cause**: Apple 的帳號變更通知機制允許攻擊者透過建立 Apple ID 並觸發帳號資訊變更通知郵件，將惡意訊息直接嵌入郵件本文中。
* **攻擊流程圖解**: 
    1. 攻擊者建立 Apple ID
    2. 攻擊者觸發帳號資訊變更通知郵件
    3. 惡意訊息被嵌入郵件本文中
    4. 郵件被發送給受害者
* **受影響元件**: Apple 帳號變更通知機制

## 2. ⚔️ 紅隊實戰：攻擊向量與 Payload (Red Team Operations)
* **攻擊前置需求**: 攻擊者需要建立 Apple ID
* **Payload 建構邏輯**:

    ```
    
    python
    # 範例 Payload
    payload = {
        "name": "John Doe",
        "email": "johndoe@example.com",
        "message": "您的帳戶已被盜用，請撥打電話至 +1-800-123-4567 進行驗證"
    }
    
    ```
    *範例指令*:

```

bash
curl -X POST \
  https://example.com/api/send-email \
  -H 'Content-Type: application/json' \
  -d '{"name": "John Doe", "email": "johndoe@example.com", "message": "您的帳戶已被盜用，請撥打電話至 +1-800-123-4567 進行驗證"}'

```
* **繞過技術**: 攻擊者可以使用 SPF、DKIM 和 DMARC 繞過郵件驗證機制

## 4. 📚 專有名詞與技術概念解析 (Technical Glossary)
* **SPF (Sender Policy Framework)**: SPF 是一種郵件驗證技術，允許發件人指定哪些 IP 地址可以發送郵件。
* **DKIM (DomainKeys Identified Mail)**: DKIM 是一種郵件驗證技術，允許發件人使用數字簽名驗證郵件的真實性。
* **DMARC (Domain-based Message Authentication, Reporting, and Conformance)**: DMARC 是一種郵件驗證技術，允許發件人指定哪些郵件驗證技術可以使用。
