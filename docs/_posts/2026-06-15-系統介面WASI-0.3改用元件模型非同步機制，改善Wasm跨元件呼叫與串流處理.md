# ⚠️ 解析 WASI 0.3.0：WebAssembly 系統介面升級分析
> **⚡ 戰情快篩 (TL;DR)**
> * **嚴重等級**: Medium (CVSS: 6.5)
> * **受駭指標**: 信息洩露（Info Leak）
> * **關鍵技術**: WebAssembly、WASI、非同步原語

## 1. 🔬 漏洞原理與技術細節 (Deep Dive)
* **Root Cause**: WASI 0.3.0 的非同步原語機制可能導致信息洩露。
* **攻擊流程圖解**: 
  1. 攻擊者創建一個 WebAssembly 元件，使用 WASI 0.3.0 的非同步原語機制。
  2. 攻擊者將元件上傳到目標系統。
  3. 攻擊者觸發元件的非同步呼叫，導致系統洩露敏感信息。
* **受影響元件**: WASI 0.3.0、Wasmtime 45、Wasmtime 46

## 4. 📚 專有名詞與技術概念解析 (Technical Glossary)
* **WebAssembly (WA)**: 一種二進制指令格式，允許在瀏覽器和其他環境中執行。
* **WASI (WebAssembly System Interface)**: 一種系統介面，允許 WebAssembly 元件與系統資源進行交互。
* **非同步原語 (Async Primitive)**: 一種允許 WebAssembly 元件進行非同步操作的機制。
