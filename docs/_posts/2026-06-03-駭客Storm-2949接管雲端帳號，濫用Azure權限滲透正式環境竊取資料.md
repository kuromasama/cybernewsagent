# 🚨 解析 Storm-2949 攻擊：雲端身分入侵與 Azure RBAC 權限濫用

> **⚡ 戰情快篩 (TL;DR)**
> * **嚴重等級**: Critical (CVSS 分數：9.0)
> * **受駭指標**: 身分入侵與雲端資源存取
> * **關鍵技術**: 社交工程、多因素驗證繞過、Azure RBAC 權限濫用

## 1. 🔬 漏洞原理與技術細節 (Deep Dive)
* **Root Cause**: 攻擊者利用社交工程手法，假冒內部 IT 支援人員，要求目標使用者核准看似正常的多因素驗證提示，從而取得使用者的 Microsoft Entra ID 帳號。
* **攻擊流程圖解**:
  1. 攻擊者發送假的多因素驗證提示給目標使用者。
  2. 使用者完成驗證後，攻擊者取得使用者的 Microsoft Entra ID 帳號。
  3. 攻擊者利用取得的帳號存取 Microsoft 365 應用程式、雲端檔案服務，以及 Azure 託管正式環境中的資料與雲端資源。
* **受影響元件**: Microsoft Entra ID、Microsoft 365、Azure RBAC

## 2. ⚔️ 紅隊實戰：攻擊向量與 Payload (Red Team Operations)
* **攻擊前置需求**: 攻擊者需要有目標使用者的聯繫資訊和社交工程手法。
* **Payload 建構邏輯**:

    ```

    # 假的多因素驗證提示

    def access_microsoft_services(user_account):
        # ...
    
    # 存取 Azure 託管正式環境中的資料與雲端資源
    def access_azure_resources(user_account):
        # ...
    
    ```
* **繞過技術**: 攻擊者可以利用社交工程手法繞過多因素驗證。

## 4. 📚 專有名詞與技術概念解析 (Technical Glossary)
* **社交工程 (Social Engineering)**: 想像攻擊者利用心理操控手法，讓目標使用者泄露敏感資訊。技術上是指利用人類心理弱點，進行攻擊和操控。
* **多因素驗證 (Multi-Factor Authentication)**: 想像使用者需要提供多個驗證因素，例如密碼、生物特徵和令牌，才能存取系統。技術上是指使用多個驗證方法，增加系統的安全性。
* **Azure RBAC (Azure Role-Based Access Control)**: 想像使用者被指派不同的角色，從而控制存取 Azure 資源的權限。技術上是指使用角色基礎的存取控制，管理 Azure 資源的存取權限。
