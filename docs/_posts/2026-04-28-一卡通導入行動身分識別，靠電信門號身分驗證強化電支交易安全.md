# ⚠️ 解析行動身分識別機制：防禦電子支付詐騙的新戰略

> **⚡ 戰情快篩 (TL;DR)**
> * **嚴重等級**: Medium (CVSS 分數：6.5)
> * **受駭指標**: 身分驗證繞過
> * **關鍵技術**: 行動身分識別、電信級身分驗證、FIDO2 無密碼驗證

## 1. 🔬 漏洞原理與技術細節 (Deep Dive)
* **Root Cause**: 傳統的簡訊OTP驗證機制容易被釣魚網站或中間人攻擊攔截，導致電子支付帳戶被盜刷。
* **攻擊流程圖解**: 
  1. 攻擊者透過釣魚簡訊或假冒客服，引導用戶點擊造假網站並輸入OTP驗證碼。
  2. 攻擊者取得用戶的帳密和OTP驗證碼，進而接管其電子支付帳戶。
  3. 攻擊者在用戶不知情下完成裝置綁定或支付工具重設，導致受害用戶的金融設定遭解除。
* **受影響元件**: 一卡通的電子支付服務iPASS MONEY，尤其是使用傳統簡訊OTP驗證的用戶。

## 2. ⚔️ 紅隊實戰：攻擊向量與 Payload (Red Team Operations)
* **攻擊前置需求**: 攻擊者需要取得用戶的帳密和OTP驗證碼。
* **Payload 建構邏輯**:

    ```
    
    python
      # 範例 Payload
      payload = {
        "username": "用戶帳號",
        "password": "用戶密碼",
        "otp": "OTP 驗證碼"
      }
    
    ```
  *範例指令*: 使用 `curl` 發送 POST 請求到電子支付服務的登入頁面，附帶 payload。
* **繞過技術**: 攻擊者可以使用中間人攻擊攔截用戶的OTP驗證碼，或者使用釣魚網站欺騙用戶輸入帳密和OTP驗證碼。

## 4. 📚 專有名詞與技術概念解析 (Technical Glossary)
* **行動身分識別 (Mobile ID)**: 一種使用電信級身分驗證技術，透過電信基地臺進行身分驗證，確認為本人操作。
* **電信級身分驗證**: 一種使用電信網路進行身分驗證的技術，透過電信基地臺進行驗證，確認為本人操作。
* **FIDO2無密碼驗證**: 一種使用公鑰密碼學的無密碼驗證技術，透過用戶的生物特徵或其他驗證方式進行驗證。
